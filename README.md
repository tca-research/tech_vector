# Tech Pulse / Tech Vector

Data pipeline and chart-generation system underlying the Tech Council of
Australia's Tech Vector dashboard (techcouncil.com.au/tech-vector). Pulls
data from a mix of automated and manually-curated sources, aggregates it,
and generates 18 standalone, self-contained chart HTML files — no chart
library, no external requests, no build step; each one just opens in a
browser (or embeds in an iframe elsewhere with auto-resize built in).

## Repo layout

```
data/
  input/
    automated_pull/   # raw pulls from WGEA, OECD, ABS, RBA, Stanford HAI, Levels.fyi
    manual_pull/      # hand-curated/uploaded CSVs — see the 3 instruction docs below
  output/             # aggregated CSVs, one per chart's data need — regenerated by automated_data_prep.py
charts/               # the 18 generated chart HTML files + chart_manifest.csv (auto-generated, don't hand-edit)
  test/               # older archived chart builds, not part of the live set
scripts/
  automated_fetch_*.py/.R   # one script per automated data source
  automated_fetch_all_data.py  # runs all 5 of the above in sequence
  manual_data_prep.py    # cleans raw manual-pull CSVs into *_cleaned.csv siblings
  automated_data_prep.py # the main aggregation step — reads data/input/*, writes data/output/*.csv
  charts/
    build_charts.py     # generates every chart HTML file from data/output/*.csv + data/input/manual_pull/*.csv
    DATA_FORMAT.md       # the data contract each chart's loader/JS expects — read before changing what feeds a chart
    assets/              # shared CSS/JS + each chart's own HTML/JS template
  MANUAL_DATA_PULL_INSTRUCTIONS_*.md  # how to refresh each manually-curated input
.github/workflows/
  rebuild-data.yml      # automated rebuild, see below
```

## Running the full pipeline locally
Please install Python and R on your system. 

```bash
pip install -r requirements.txt
# R packages (one-time): Rscript -e 'install.packages(c("readabs", "readrba"))'

python3 scripts/automated_fetch_all_data.py            # refresh data/input/automated_pull/*
python3 scripts/manual_data_prep.py --dir data/input/manual_pull/   # clean any new manual-pull CSVs
python3 scripts/automated_data_prep.py                  # aggregate everything into data/output/*.csv
python3 scripts/charts/build_charts.py                  # regenerate charts/*.html
```

Each step is independent and safe to re-run — `build_charts.py` in
particular only bumps a chart's "last updated" date (both on-chart and in
`chart_manifest.csv`) when its underlying data actually changed, not on
every run.

## Automated rebuild (GitHub Actions)

`.github/workflows/rebuild-data.yml` runs the same 4 steps above on an
**8-week cycle** (anchored to 2026-08-10) and commits the result straight
back to `main`. GitHub Actions cron can't express "every 8 weeks" directly,
so it actually runs weekly and gates itself on a day-count check — see the
workflow file's comments for the exact math. Trigger it manually any time
via Actions → "Rebuild data and charts" → Run workflow → `force: true`,
without waiting for a real cycle boundary.

This automation **only handles the automated-pull sources** — it does not
and cannot refresh the manually-curated inputs (see below). There is
currently no reminder mechanism or deployment step (e.g. GitHub Pages) —
both were scoped out for now; if you want either, the workflow already has
a clean seam to add a `notify`/`deploy` job onto.

## Manually-curated data

Three inputs need a human to refresh them periodically — nothing in this
repo can fetch them automatically. Each has its own instructions doc:

- **`scripts/MANUAL_DATA_PULL_INSTRUCTIONS_HEADLINE_METRICS.md`** — the
  Tech Investment / Tech Sector GDP gauge figures (`headline_metrics.csv`)
  and the Tech Jobs gauge's raw source (`tech_jobs_in_australia.csv`).
- **`scripts/MANUAL_DATA_PULL_INSTRUCTIONS_TECH_COUNCIL_ABNS.md`** — the
  current list of Tech Council member ABNs (`Tech_Council_ABNs.csv`), used
  to identify which WGEA-reporting companies are TCA members.
- **`scripts/MANUAL_DATA_PULL_INSTRUCTIONS_TABLEBUILDER_UPLOAD.md`** — ABS
  TableBuilder exports for the tech-roles-by-subsector breakdown.

## The chart system

Every chart is config-driven: title, caption, source line, column mappings,
series/colors, and download filename all live in a `CONFIG` dict in
`scripts/charts/build_charts.py` — none of it is hardcoded in the shared
JS/CSS in `scripts/charts/assets/`. Repurposing a chart for a different
dataset with the same shape (a category + N series columns, for example)
is usually just a new config, not new code.

Read **`scripts/charts/DATA_FORMAT.md`** before changing what feeds any
chart, or adding a new one — it documents the exact JSON shape each chart
type expects, the ISO-date rule (getting this wrong renders a chart with no
error and a subtly wrong x-axis), and a few other non-obvious gotchas
(gauge met/not-met unit matching, the `{year}` source-line convention, the
"only stamp a new date when data changed" mechanism).
